# Frontend Agent Instructions

## Overview
This agent specializes in React/TypeScript frontend development for the GoBarber web application.

## Project Structure
```
gobarberweb/
├── src/
│   ├── components/     # Reusable UI components
│   ├── pages/         # Route components
│   ├── hooks/         # Custom React hooks
│   ├── services/      # API integration
│   ├── styles/        # Global styles
│   ├── utils/         # Helper functions
│   ├── types/         # TypeScript definitions
│   └── routes/        # Route configuration
├── public/            # Static assets
└── .storybook/        # Storybook configuration
```

## Technologies
- **React 18** with TypeScript
- **Styled Components** for styling
- **React Router** for navigation
- **Axios** for API calls
- **Unform** for form handling
- **Yup** for validation
- **Polished** for style utilities
- **Storybook** for component development

## Coding Standards
- Use functional components with hooks
- TypeScript strict mode enabled
- Styled components for component styling
- Custom hooks for shared logic
- Error boundaries for error handling
- Responsive design principles

## API Integration
- Base URL: `process.env.REACT_APP_API_URL || 'http://localhost:3333'`
- JWT authentication via localStorage
- Automatic token refresh handling
- Comprehensive error handling

## Component Patterns
- Container/Presentational pattern
- Custom hooks for business logic
- Props interfaces for type safety
- Default props where appropriate
- Accessibility considerations

## Development Workflow
1. Create components in `/components`
2. Add stories in Storybook for testing
3. Use hooks for state management
4. Implement proper error handling
5. Test responsive design
6. Follow TypeScript best practices

## Key Components
- **Button**: Loading states, variants
- **Input**: Icons, validation, error states
- **Toast**: Notifications system
- **Tooltip**: Hover information
- **Auth**: Authentication flow

## Performance Considerations
- Lazy loading for routes
- Memoization for expensive operations
- Optimized re-renders
- Bundle splitting
- Image optimization
